# IT Service Management (ITSM) Dashboard | Power BI

## Overview

This project presents an interactive **IT Service Management (ITSM) Dashboard** developed in **Power BI** to monitor service desk performance and provide actionable insights into ticket management operations.

The dashboard enables stakeholders to track ticket inflow, monitor resolution performance, analyze ticket categories, and identify operational trends that support data-driven decision-making.

## Business Problem

Service desk teams handle a large number of Incidents, Service Requests, and New Requirements every day. Without proper reporting, it becomes difficult to monitor workload, identify bottlenecks, and evaluate team performance.

This dashboard consolidates ITSM data into an interactive report that helps managers monitor key operational metrics in real time.

---

## Key Performance Indicators (KPIs)

### 📈 Ticket Inflow Trend
- Daily ticket creation trend
- Overall ticket volume over time
- Incident vs Request distribution
- Trend analysis for increasing or decreasing ticket inflow

---

### 🎫 Ticket Type Analysis
- Incidents
- Service Requests
- New Requirements
- Comparative ticket volume by type

---

### ✅ Resolution Status Analysis
- Assigned Tickets
- Work In Progress (WIP)
- Reopened Tickets
- Closed Tickets
- Resolved Tickets

Provides visibility into ticket lifecycle and operational efficiency.

---

### 📊 Resolution Performance
- Resolution percentage
- Closure rate
- Pending workload
- Till-date ticket status summary
- Period-wise ticket resolution analysis

---

### 📂 Open/WIP Category Analysis
Analyze workload distribution across categories such as:

- Functionality
- Configuration
- Dashboard
- Data
- Integration
- Others

Helps identify areas generating the highest number of unresolved tickets.

---

### 📅 Period Analysis
Compare ticket performance over selected reporting periods to identify:

- Daily trends
- Weekly performance
- Operational workload
- Resolution efficiency

---

## Insights Generated

The dashboard helps answer questions such as:

- How many tickets are created each day?
- Is ticket inflow increasing or decreasing?
- What percentage of tickets have been resolved?
- How many tickets remain assigned or in progress?
- Which categories generate the highest number of tickets?
- What is the overall service desk performance?
- How effectively are incidents being resolved?
